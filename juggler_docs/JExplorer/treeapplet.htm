Using the JExplorer/JFinder

Version 1.08 - 11/1/2000
To use jexplorer you must first include the applet tag in your web page (see using applets below) including a file1 param tag which points to a text file specifying the items and there relationships. The items are laid out in an XML like format defined in the paramfile.htm.
Note: You must keep all the dot class files distributed with the package in the same directory and keep the filenames identical (including case) otherwise the applet will be unable to start.
file(n) = URL
File is the only required parameter of JExplorer. File specifies the initial parameter file that is loaded on initialization and any other files that can be selected by the javascript interface setFile. Files are numbered sequentially (file1, file2 .. fileN) each specifying the URL of a file to be loaded. The URL's for the files are relative to the html document base from which the applet is loaded and are text files laying out items in the parameter file format specified. The default file1 URL is the base html page URL.
bgImage = URL
bgImage defines the URL of a background image to display in the background of the applet. The URL is relative to the applets URL or maybe defined with a full URL (see Using URL's ). The image maybe in gif or jpg format and is drawn in the top left hand corner of the applet background and tiled if tile keyword is included in the pramater list.
tile
Tile background image if set will tile the background image across the applets background. Tile is a keyword and needs no arguments.A value argument is required under Netscape as the keyword is removed if it not present. The actual value argument is not processed so its value is not important.
showURL
If defined the URL of an item is appended to the info text string of the item. This is useful for debugging as the full URL can be seen and checked. ShowURL is a keyword and needs no arguments.A value argument is required under Netscape as the keyword is removed if it not present. The actual value argument is not processed so its value is not important.
hidestatus
If defined no text will be displayed on the status bar when items are selected.
hidelines
Removes all lines from between the items displayed.
autoclose
Autoclose is an optional folder mode. When selected any folder that is opened will automatically close any other folder on the same level. The is option is turned off by default though the autoclose option is useful for conserving vertical space in the applet and keeping the display clear of unnecessary items. Autoclose is a keyword and needs no argruments.A value argument is required under Netscape as the keyword is removed if it not present. The actual value argument is not processed so its value is not important.
bgColor = color
This is the color that the background of the applet is filled with before any background image is drawn. The default color is white.
fgColor = color
fgColor or foreground color is used to draw the lines between items. The default color is black.
textColor = color
The text color is used to draw the individual item titles. The default color is black.
bgHighlight = color
When an item is selected the item is redrawn with a highlighted background color. behind the text of the item. The bgHighlight color is by default transparent but may be set to any color.
textHighlight = color
The text highlight color is used when an item is selected and the item is redrawn with the highlight icon and background highlight colors. By default the highlight text color is lightGray.
mouseOver= color
When the mouse is over an item it is redrawn using its highlight image and the mouseover color for the text.  The default color is blue.
mouseunder = color
When the mouse is over an item a line is draw under the selected text of the item. Mouseunder specifies the color of this line. The default color is blue.
tooltext = color
Tooltext is the text color of tooltip or popup info text of an item. The default color is blue.
toolback = color
Toolback is the background color of the popup tooltips or popup info boxes. The default color is white.
plusColor = color
Plus color is the color of the folder expand and collapse +/- signs . The default color is lightGray or the same as fgColor if fgColor is defined.
barsize = int
barszie defines the width in pixels of the scrollbars. This width is by default 12 pixel.
title = text
Title is the title that is displayed as the applet loads a file from the host server. It can be any text string but defaults to www.jexplorer.com if not defined. The text may bee null "" in that case nothing is displayed.
Example
<applet width="100%" height="75%" name="Tree" code="TreeApp.class" archive="tree.zip" codebase="./" align="top">

Load three files all in applet directory
<param name="file1" value="test.txt">
<param name="file2" value="test2.txt">
<param name="file3" value="test3.txt">

Set applet colors
<param name="bgHighlight" value="rose">
<param name="textHighlight" value="white">
<param name="mouseover" value="blue">
<param name="mouseunder" value="0A0A0A">

Add a background image and tile it
<param name="bgImage" value="tile.gif" >
<param name="tile" value="" >

<param name="title" value="The JExplorer" >

And show the URL of items in info text
<param name=showURL" value="" >
</applet>

Using Colors
Colors can be defined in two ways. Firstly a color can be described in short hand from a range of predefined colors or the color can be defined exactly using a HEX constant. If a color is not specified or its invalid the control will use the default color.
Predefined colors.
A range of shorthand colors are available and are used by name see the example below. The colors available are black, blue, cyan, darkgray, gray, green, lightgray, magenta, orange, pink, red, white, yellow or rose.
Hex Constant Colors
In this format, the red, blue, and green components of a color are each represented by an integer in the range 0-255 ( 00 to FF ). The value 0 indicates no contribution from this primary color. The value 255 indicates the maximum intensity of this color component. For example the color white would be FFFFFF the three components being equal to full intensity FF or 255 respectively. The color red would be FF0000 and blue being 0000FF. 
Example
<param name=bgColor value="black">

<param name=bgHighlight value="lightgray">

<param name=fgColor value="A0A0A0">

<param name=textColor value="white">

<param name=textHighlight value="FF00A0">

Using Applets
< applet codebase = URL code=applet-filename width= pixel-width height = pixel-height name=applet-name archive=archive-file>
</applet>
Codebase
The is optional attribute specifies the base URL of the applet. This is the directory and not the applet file itself. If not specified then the URL of the current document is used.
Code
This is the location of the complied applet or class file and is defined relative the codebase or the current document if no codebase is defined.
Width and Height
This is the width and height of the applet in pixels or in percentage. If defined as a percentage the width and height of the applet may change as the page is resized. This is true for IExplorer as the applet is notified of the change in size. Under some versions of Netscape the sizes are changed but the applet is unable to respond.
Name
This is the name that the Java object will be given. Using JavaScript this will be object name of which calls can be made.
Archive
To speed up the initialization time of Java applets newer browsers include the option of downloading the resources needed from a single archive or zip file.The filename is relative to the codebase and should contain the class files required to run the applet.  The archive file is included with the distribution classes.
