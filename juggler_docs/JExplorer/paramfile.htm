JExplorer/JFinder parameter file format

Version 1.08 - 11/1/2000
The treecontrol parameter file defines items to be added to the tree. The format of the parameter file is close to the XML tag specification but does not include all the features of the true XML. The treecontrol parser ignores all tags other than the ones defined and any information outside tags so other information such as notes and server side tags may also be included within the parameter files. You may only define one treecontrol block within each file as the parser will not go looking for further treecontrol blocks. Errors that occur during processing are attached to the items info text so may be viewed on the status bar or by the ToolTips popup box.

TreeControl
< treecontrol linkurl="URL" imageurl= "URL" selectURL="URL" selectID=int>
< /treecontrol >
Treecontrol is the block tag in which all the tree control tags have to be defined. The applet will read a file until the treecontrol tag is found and process items until it encounters the end block ‘/treecontrol’ tag. This allows the file to hold other information and formatting out side of the tree control block. A treecontrol block may not have child treecontrol blocks and the inclusion of such will have undefined results.
linkurl
The linkurl defines the base URL for all links and jitfolder links defined within the file. Linkurl is combined with the page URL to which the applet belongs inheriting any components that are not defined within the link URL. ( see Using URL's for more information ). Any links attached to items are in turn combined with this URL as a base. The major use for this is to make all links relative to one base so when you move the files you need only change the base URL and not each individual link.
imageurl
The imageurl is similar to the linkurl as it defines the base URL for all images referenced by individual items. The imageurl is combined with the pagebase URL in the same manor as the linkurl. This means that images are referenced by default from the URL position of the html file that includes the applet. (see Using URL's for more information ). Any images attached to items are in turn combined with this URL as a base.
selectURL
SelectURL sets a particular item into the highlighted state after the script file has been loaded. This is done by selecting the first item found with a corosponding URL. If selectID is used aswell the ID is selected in priority to the URL. Note that any search is done across all items not just the ones loaded in the current script file.
selectID
SelectID sets an item into the highlighted state after the script file has been loaded. This is similiar to the selectURL argument with the item selected having a corosponding ref number attached (see item). Note that any search is done across all items not just the ones loaded in the current script file.
Example
< treecontrol linkurl="http://www.mcsoftware.com.au/java" imageurl=".\images\" selectURL="http://www.jexplorer.com" selectid=6>
All links are based at the fully defined directory on mcsoftware.com.au at the directory java and images are in the directory images under the calling html document.

< /treecontrol > Close treecontrol block

Item
< item title="text" image="URL" imagetwo="URL" link="URL" rlink ="URL" target="frame" rtarget="frame" width="int" height="int" info="text" >
The item tag defines an item within the tree structure. The position of the item depends on its position within the parsed file. All item tags must appear between the treecontrol block tags. The item will be added to any enclosing folder if defined within the folder block tags. If there is no enclosing folder it will be added to the root of the tree structure. The font used by the item depends on the font block which it belongs by default the font of the parent folder is used.
title
This is the text that will appear as the item label. An item may have spaces or any printable character in its title. If the title is undefined it takes the title of the idefault which is "default" by default. You may make an item have no text label by putting no text between the double quotes or have the same text as another item. Items can have only one line of text. The inclusion of any newline characters will have undefined results.
image
Image defines the URL of the unselected icon for this item. The icon should be in GIF or JPEG format and may be of any size although icons greater that 30 by 30 pixels have not be tested and may have undesired effects. But there is no reason that larger icons cannot be used. The image icon is optional if no image is defined or there is a error finding or loading the desired image then only the text label will appear
imagetwo
Imagetwo is the URL of the image displayed when the item is highlighted or selected. This icon should be of similar dimensions to the image icon to avoid undesired shifting when selected. The size of the item is computed from the width and height of the largest icon with the width of the text being added and the height of the text taken if larger. The imagetwo icon is optional if no icon is defined then image icon is used when the item is selected.
link
Link is the URL passed to the browser on left mouse button selection of the item. It may be a relative link (see URL’s) or include a full http:// ftp:// mail:// prefix’s. The browser frame may be selected using target. Link is optional if no URL is defined the link equals none or is of no length, then no action will occur on left mouse button events. The full left mouse button URL appears on the status bar and appended to the popup status information when ‘showURL’ is included within the applet param list. This can be useful for debugging and as an aid to clients. See the applet documentation to read more.
rlink
Rlink is the URL passed to the browser on right mouse button selection of the item. It is included as people wanted to add an optional properties page for each link but may have other uses. It is defined in the same way as link. Rlink is optional if no link is specified then the link URL is used on right mouse button events. Rlink is used in conjunction with rtarget as the target browser frame.
target
This is the target frame to which left and right mouse links are displayed in. The default is the browser default "_top" but may be any relevant frame name. This is included separately from the links as it may be defined within a idefault block and save space and processing time.
rtarget
Rtarget or right mouse target is used and specified in the same manor as target. rtarget is the target browser frame for right mouse button links. If not specified it will default to the same frame as target which in turn defaults to the browser default which is usually set to "_top". By setting rtarget to _blank and not specifying a rlink you can get normal links to appear in an optional new window when the right mouse button is used.
info
Info is the text that appears on the status bar of the browser window when the mouse is over this item. This text will also be included in the popup status information when the mouse hang is detected. The full URL of the item will appended to the info text if ‘showURL’ param is defined in the applet argument list. . See the applet documentation to read more. Errors encountered in processing or loading certain items will be reported in the info message text.
width
Width is the minimum width in pixels an item can occupy. If the item is bigger than the default minimum it will expand to fill the area required.
height
Height is the minimum height in pixels an item can occupy. If the font or image require more space the item will be resized to fit
Example
< treecontrol > Open treecontrol block

A simple example
<item title="Example Item" info="An example item with no image or links" >

Another example with images and links
<item title="Another Example"  info="Short information string" image="doc_web.gif" imagetwo="doc_web2.gif" link="mypage.html" rlink="http://www.mcsoftware.com.au/docs/about.html" target="main" >

< /treecontrol > Close treecontrol block

Folder
< folder expand title="text" image="URL" imagetwo="URL " link="URL" rlink ="URL" target="text" info="text" arrow1="image" arrow2="image" arrow3="image" >
< /folder >
Folder encloses a block of items. Any items that appear between the opening and closing folder tags will be added as a child item of the folder. Folders may contain other folders. Each folder block must be closed with the corresponding ‘</folder>’ tag.
expand
Display the folder in an initial expanded state. Any folder may be expanded by default and is not dependent on its depth or whether it is initially visible. Expand is a keyword and has no arguments including it in the parameter list is enough. A folder is expanded and collapsed by a double click or a click on the ‘+’ or ‘-‘ a single click will activate any link URL attached to the folder the same as an item.
arrow1, arrow2, arrow3 (JFinder only)
Arrow1, arrow2 and arrow3 specify the images used in JFinder as the open and close folder arrows on the left of the folder item. Arrow1 is the default folder closed arrow image. Arrow2 is the itermediate opening arrow and arrow3 is the openned folder arrow image. These three images may be different sizes and may be gif or jpg images and are loaded on initalisation the size is taken from the biggest image. You may specify only one two or all three images. Missing images will not be shown.
You may specify different images to different folders to give different look and feel. But for most applications use an idefault block to define the open and close images for all folder items.
The rest of the parameters are inherited from item. See item for more information.
Example
< treecontrol > Open treecontrol block
<item title="Example item outside folder" >

Create a folder
<folder title="Simple Folder" image="fold_grey.gif" imagetwo="fold-grey2.gif" arrow1="f1.gif" arrow2="f2.gif" arrow3="f3.gif">
   <item title="An item of simple folder" >
</folder> And close folder block

< folder title="An open folder" expand image="fold.gif" imageTwo="fold2.gif" >
    < item title="Items with the same text" image="mail.gif" >
    < item title="Items with the same text" image="mail.gif" >
    < item title="nextItemOne" info="hi yet another info string">
    < item title="nextItemOne" info="more information about mcsoftware">
    < item title="nextItemOne" info="more information">
    < item title="" info="An item with no text" imagetwo="o_folder.gif" >
    < item title="nextItemOne" >
</folder> And close item
< /treecontrol > Close treecontrol block

Jitfolder
< jitfolder loadasroot jit="URL" title="text" image="URL" imagetwo="URL" link="URL" rlink ="URL" target="text" info="text" >
A jit or just in time (borrowing from somewhere :-)) is a folder that is loaded as needed. It appears as a regular folder within the tree but its child items are loaded from file when the folder is expanded. The loading process goes on in the background so it doesn't’t lock up the control. The use of jitfolders can be useful where a implementation has a large number of items that may not be needed by every client browser. The arguments for the jitfolder describe how it will be displayed within the current tree structure but may be modified by the loaded file using the rootfolder tag. The current font for the jitfolder will be the default font for the loaded child items.
jit
URL of a treecontrol parameter file. The URL inherits from the link URL defined in the treecontrol tag so can be relative to the link URL ( see URL's and treecontrol tag for more information). A JIT file URL must be included or it may result in undefined actions. Any errors with the loading a Jit folder will be reported in the info message attached to the item. To read this leave the mouse of the item and read the popup status message.
loadasroot
Loadasroot is a keyword that will force the loading JIT file into as the root folder. This can be used (especially if you have a large number of links in the applet) to discard all the items above this folder and expand this folder as the new root level directory. 
The rest of the  parameters are inherited from item. See item for more information.
Example
< treecontrol > Open treecontrol block

A simple folder with items loaded as needed
< jitfolder  jit="/java/treefile.txt" title="JIT Folder" image="fold_grey.gif" imagetwo="fold-grey2.gif >

Create a folder
<folder title="Simple Folder" image="fold_grey.gif" imagetwo="fold-grey2.gif>

A child folder with items loaded from file test.txt as needed
< jitfolder  jit="test.txt" title="Simple child folder" image="fold_grey.gif" imagetwo="fold-grey2.gif >

</folder> And close folder block
< /treecontrol > Close treecontrol block

 IDefault
< idefault title="text" image="URL" imagetwo="URL" link="URL" rlink ="URL" target="text" info="text" > </idefault>
idefault sets up the defaults for created items and folders. All items within the idefualt block inherit the settings of the idefault item. This is useful for setting a series of items with similar arguments and can save having to modify a large number of similar items. Items can define any of there own arguments but any missing arguments will be inherited from the idefualt item. Idefault tags can appear anywhere within a treecontrol block and only affect items within the idefault block.
All parameters are inherited from item. See ‘item’ for more information.
Example
< treecontrol > Open treecontrol block

< idefault title="Another Item" image="doc_open.gif" rtarget="_balnk" >

    < item > This item will inherit all the settings of idefault
    < item title="A different item with the same images" >
    < item image="doc_close.gif"> Item with a different image
    < item >And the default item again

< /idefault > And close the idefault block

< /treecontrol > Close treecontrol block

Font
< font face="text" size="int" style="text" >
< /font >
Font defines a block within which all the items will be set to the specified font. Any or all of the arguments can be defined. So setting just the size argument will take the current font and modify the size. Fonts for a file are inherited from its parent folder the default initial font being ‘dialog 10 plain’.
face
Any valid font face. A valid font face name depends on the platform you are running on but widely supported standard fonts are Helvetica, TimesRoman, Courier and Dialog. The font names are case sensitive and can include spaces.
size
Size of the font any valid font size. The default is size 10.
style
Styles can be plain, italic, bold or bolditalic.
detailcolor = color
detailcolor or foreground color is used to draw the lines between items. The default color is black.
color = color
The text color is used to draw the individual item titles. The default color is black.
bghighlight = color
When an item is selected the item is redrawn with a highlighted background color. behind the text of the item. The bgHighlight color is by default transparent but may be set to any color.
texthighlight = color
The text highlight color is used when an item is selected and the item is redrawn with the highlight icon and background highlight colors. By default the highlight text color is lightGray.
mouseover= color
When the mouse is over an item it is redrawn using its highlight image and the mouseover color for the text.  The default color is blue.
mouseunder = color
When the mouse is over an item a line is draw under the selected text of the item. Mouseunder specifies the color of this line. The default color is blue.
plusColor = color
Plus color is the color of the folder expand and collapse +/- signs . The default color is lightGray or the same as fgColor if fgColor is defined.
Example
< treecontrol > Open treecontrol block

< folder title="Fonts" image="fold_grey.gif" imageTwo="fold_grey2.gif" >
< font face="Courier" >
    <font style="plain">< item title="Courier" ></font>
    <font style="plain">< item title="plain" ></font>
    <font style="bold">< item title="bold" ></font>
    <font style="bolditalic">< item title="bolditalic" ></font>
    <font style="italic">< item title="italic" ></font>
</font>

< folder title="Font sizes" image="fold_zoom.gif" imageTwo="fold_zoom2.gif" >
< font face="TimesRoman" style="plain" size="8">
    <font size =8 >< item title="Times Roman size 8" > </font>
    <font size =10 >< item title="Times Roman size 10" > </font>
    <font size =12 >< item title="Times Roman size 12" > </font>
    <font size =14 >< item title="Times Roman size 14" > </font>
    <font size =16 >< item title="Times Roman size 16" > </font>
    <font size =18 >< item title="Times Roman size 18" > </font>
   <font size =20 >< item title="Times Roman size 20" > </font>
   <font size =22 >< item title="Times Roman size 22" > </font>
   </font> End TimesRoman
</folder> Close folder Font Sizes
</folder> End Folder Fonts
< /treecontrol > Close treecontrol block

Root folder
< rootfolder title="text" image="URL" imagetwo="URL" link="URL" rlink ="URL" target="text" info="text" >
The rootfolder tag gives access to the root folder into which the treecontrol file is being loaded. If this is the first file loaded this will be the folder at the top of the control but may be any jit folder. The rootfolder may appear anywhere within the treecontrol block and should only be defined once. You may set any of the parameters of the parent folder including changing the icon. The rootitem parameters will be affected by the idefault settings if placed within a idefault block
All parameters are inherited from item. See ‘item’ for more information.
Example
< treecontrol > Open treecontrol block

Set root folder text and images
< rootfolder title="SiteMap" image="fold.gif" imagetwo="fold2.gif" info="A site map of pages on mcsoftware.com.au">

Add some items to the tree.
< item image="tp_up.gif" >
< item image="tp_down.gif" >
< item image="tp_stop.gif" >

< idefault title="An Item" image="doc_open.gif" >
   < item image="tp_rewind.gif" imagetwo="tp_ff.gif" >
   < item image="tp_play.gif" >
   < item image="tp_eject.gif" >
   < item image="tp_back.gif" >
</default>

< /treecontrol > Close treecontrol block

Using URL’s with the jexplorer.
A URL represents a Uniform Resource Locator—a pointer to a "resource" on the World Wide Web. A resource can be something as simple as a file or a directory, or it can be a reference to a more complicated object, such as a query to a database or to a search engine. More information on the types of URLs and their format can be found at:

http://www.ncsa.uiuc.edu/demoweb/url-primer.html

In general, an URL can be broken into several parts. The above URL indicates that the protocol to use is http ("HyperText Transport Protocol"), that the information resides on a host whose name is www.ncsa.uiuc.edu. The information on that host machine is named demoweb/url--primer.html. The exact meaning of is name on the host machine is both protocol- and host-dependent. The information could reside in a file or could be generated on- the-fly.

A URL can optionally contain a "port," which is the port number to which the connection is made on the remote host. If the port is not specified, the default port for the URL is used instead. For example, the default port for http, is 80. An alternative port could be specified as:

http://www.ncsa.uiuc.edu:8080/demoweb/url-primer.html

A URL may have appended to it an "anchor", which is indicated by the sharp sign character "#" followed by more characters. For example

http://local/demo/information#myinfo

This is not technically part of the URL. Rather, it indicates that after the specified "resource" is retrieved, the application is specifically interested in that part of the document that has the tag "myinfo" attached to it. The meaning of a tag is resource specific.

An application can also specify a "relative URL", which contains only enough information to reach the resource relative to another URL. Relative URLs are frequently used within HTML pages. For example, if the URL

http://java.sun.com/index.html

contained within it a reference to the URL "FAQ.html", it would be a shorthand for

http://java.sun.com/FAQ.html

The relative URL need not specify all the components of a URL. Missing components are inherited from the fully specified URL.

All URL’s with the treecontrol applet are relative to the two base URL's defined in the treecontrol tag. Which are in turn relative to the URL of the html page the applet is called from.

Due to Java security mechanism the treecontrol applet may not load any components from any machine other than the one it resides on. This doesn't’t affect links handed to the browser such as loaded pages but does effect loaded treecontrol parameter files and images as these are processes directly by the applet.
