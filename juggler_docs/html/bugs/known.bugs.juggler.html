Juggler Buglist

Ver 0.1.95

  • LiNUX runtime reconfiguration
    - not stable
  • Win32 Input Device Drivers
    - input device drivers not built in Win32 version. (serial port/socket issues)
  • Display windows can't be resized with mouse
  • "Combo" Sample application is broken
    - Has a problem in its makefile
  • "Cubes" Sample Application
    - the cubes don't display in Win32 version of Juggler
  • Runtime reconfiguration of Performer components
    - implementation is not complete
  • VjControl cannot connect to Win32 Juggler apps
  • Glove device drivers
    - untested
  • Memory leaks
    - Has just a few minor ones ;)
  • VjConfig files
    - path names must be absolute
  • LiNUX device drivers
    - not well tested
  • Runtime reconfiguration
    - there are a few things that can not be reconfigured (however, most display and input things do work well)
  • Control-C to exit
    - no elegant way yet
  • juggler/test/CfgTest sanity checking for config code
    - broken

Beta 1

  • Killing the sample applications requires a Ctrl-C.  This is because there is no device that is guaranteed to always be active.  In a future update, the GUI will allow you to kill a running application.
  • HMD support is not included in this beta.
  • Performer runtime reconfiguration support is currently broken.
  • Performer stereo is broken.
  • The only tested glove device is the simulator glove device.
  • Gesture recognition for non-simulated gloves is not supported.
  • The immersion box driver does not recognize the port property.
  • Removing a keyboard device crashes the system.
  • String values with quotes are not allowed in the config system
  • Config includes require absolute paths
  • It is possible to dynamically reconfigure the system in such a way as to remove something that another object is dependant upon.  This has undefined results... but usually it is very bad.
  • It is beta software, so there are many memory leaks but there are no large memory leaks that we know of.
