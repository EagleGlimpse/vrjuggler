VRJuggler Workshop Part I

    Overview of Juggler and its capabilities

      Purpose of Juggler:

      • To allow you to create VR applications as easily as possible.
      • To allow you to create an application that runs on many different displays (C2, C6, head-mounted, 3d auditorium, etc.) and also allow you to create and test it with the same tools.
      • To allow you to hook up different interfaces, such as a wand or a glove, without having to write the drivers yourself.  You'll be able to change configuration files with vjControl.
      • To acheive optimal performance when running your VR application.

      Extras that will help you create VR applications.

      • A simulator -- because no one person has access to an expensive CAVE 24 hours a day.
      • You don't have to worry about setting up displays, double-buffering, etc.  All that is done for you with optimizations in mind.
      • Helpful math libraries such as vector, matrix, and quaternion classes are included, which you can use or include your own instead.

    Running a simple program in the simulator

      Compiling

      Problems compiling samples may be caused by incorrect variable and path setup.

      Simulating

      Enter the executable name followed by a list of configuration files.
      Example:  program_name ~/.vjconfig sim.base.config ~/.vjconfig/sim.wand.mixin.config

      What you see

      Three small windows and one large window should appear.  The large window is your rendering window.  The small small windows are for controlling the head, wand, and camera (camera is only to help with simulation, and will not appear when running in your virtual environment.

      Interaction/Control

      Move the mouse over a window and use the buttons that are defined in your sim config file.
      The default control for the camera and head are located on the number keypad.  To move the wand, hold down an alt, control, or shift key while moving the mouse in the mouse control window.
      The configuration is specified in your config files and can be changed with vjControl.

    vjControl

    Has its own documentation section on this website.
    Additional requirements: set path to include $VJ_BASE_DIR/bin and setenv JDK_HOME /usr/java)

      Purpose

      to allow the same application to be set up for different input/output devices.  Allows it to change at run-time (without recompiling).

      Run vjControl

      Type "vjcontrol".  If there is an error make sure your paths and environment variables are set correctly.  If you're having problems, you're version of java may be 1.1.  Try running "vjcontrol1.1" instead.

      Predefined configuration files are available.

      When starting to learn juggler it is not necessary to modify configuration files.  You will want to use vjControl after you have  created basic Juggler programs with the default configuration files.

      Modify a configuration file yourself:

      • If you want to change something, try using existing config files as a starting point.
      • Simple vjControl operations: Remove, Copy, Insert.
      • Try changing something simple:
      • In sim.base.config:  change the keys in Input|Simulated Devices|SimCamera1

        • Or try adding a fourth button to the wand in Input|Proxies and Aliases|DigitalProxy

      Summary

      • Not absolutely necessary for beginners, but you will want to learn how to do it.
      • If you don't like the current interfaces like the simulator, you can modify some of it.  If you are using custom peripherals you will have to customize it.

    A more in depth look at how Juggler works

      Function uses and descriptions

      To find out more about each function, look in the Programmer Reference Book on this website.

      Basic flow of a vrJuggler program.

      There are more functions (ContextPreDraw, PipePreDraw, etc.) but learn the following first:

      init();          // Init before API is started
      api_init();      // Init after API is started
      contextInit();   // (OpenGL specific) Init, do things like create display lists.
      while (drawing)   // The Drawing Loop
      {
          preFrame();
            draw();
                intraFrame();      //   Drawing is happening while here
                  // CAREFUL: don't modify data in intraFrame that is used in the draw function.
                  // if you want to do processing that data here, you must buffer it.s
          sync();
              postFrame();      // Drawing is now done
          UpdateTrackers();
      }

      Note: Performer applications have some different functions, but the idea is mostly the same.  You can refer to the header files in Programmer Reference for specific information.
