Developing Applications for VR Juggler

There are many things to learn about developing for Juggler, but there are a few key concepts that you need to keep in mind as you read the documentation:

All documentation is Online!

There are no books, or README files that you have to wade through.
Most new developers will use the "Programmer Guide" to get started, and to understand new concepts and conventions that Juggler uses.
When you begin writing code you will need to use the "Programmer Reference", which outlines everything in the juggler library.

Your application is a class object

All VR Juggler applications are user defined C++ classes.  These classes are derived from abstract base classes defined by VR Juggler.  The user defined applications extend the base classes by defining implementations of the base class virtual functions.

Different Application Base Class for each API

There are base application classes for each API.  These classes provide the interface that Juggler needs for that specific API.  There are also API specific extensions for some base classes to give users extra abilities or make programming for some specific feature of that API easier.
