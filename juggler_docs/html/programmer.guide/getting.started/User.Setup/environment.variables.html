Environment Variables

Introduction

A number of environment variables affect the way VR Juggler works. Some of these are required to compile and run applications, while others are optional. This page lists all of these environment variables.

The syntax for setting or changing an environment variable varies with operating systems and shell interpreters. This page assumes the syntax used by C-style shells (for example, csh or tcsh). Please refer to the documentation for your shell for more information about how to set, reference and modify environment variables.

Note for Win32 users using DOS shells: The typical syntax for setting an environment variable from the command line under Windows is: 

    > set VARIABLE_NAME=value
Here, VARIABLE_NAME is the name of the environment variable to be set. To view the values of environment variables, just enter set; for the value of a single variable, enter set VARIABLE_NAME. For more information, refer to the Windows online documentation about environment variables.

Required Environment Variables

VJ_BASE_DIR

The environment variable VJ_BASE_DIR tells a VR Juggler application where to find important data files, and is required to compile or run any Juggler app. It should be set to the base directory of the installed juggler library.

For example, if you downloaded a Juggler tar file and extracted it to the directory /home/software/, you would set VJ_BASE_DIR with this command:

ex. setenv VJ_BASE_DIR /home/software/vrjuggler-0.1.95

The last component of the above example depends on the particular version of Juggler you've downloaded.

If you downloaded and built VR Juggler from the source code, the compilation creates a directory called instlinks which can be used as a Juggler base:

ex. setenv VJ_BASE_DIR ~/my_build_dir/instlinks

In either case, on a Win32 platform, you should use /'s as the path separator for VJ_BASE_DIR rather than \'s. The compiler tools can handle either, and the utilities in juggler-tools will behave much better if UNIX-style paths are used. It is safe to use the drive letter at the start of the path (e.g., ``C:/software/vrjuggler-0.1.95'').

JDK_HOME

The JDK_HOME environment variable is required by the script that starts VjControl, the VR Juggler configuration program. If Java is installed on your system, JDK_HOME may already be set. If not, it needs to be set to the base of the Java installation.

Ex. setenv JDK_HOME /usr/java

LD_LIBRARY_PATH (UNIX/Linux only)
LD_LIBRARYN32_PATH (IRIX only)
LD_LIBRARY64_PATH (IRIX only)

Unix/Linux systems use these environment variables to find dynamically-loaded libraries, such as libJuggler.so. Unless you're building everything with static libraries, you will need to set these to include the Juggler library directory (under $VJ_BASE_DIR).

NOTE: if the LD_LIBRARY_PATH is already defined, you do not want to destroy the current value, just append the new path to it. Otherwise something else could stop working.

ex. setenv LD_LIBRARY_PATH $VJ_BASE_DIR/lib

ex. setenv LD_LIBRARY_PATH $VJ_BASE_DIR/lib:$LD_LIBRARY_PATH

NOTE: On some SGI systems, users of the MIPSpro compiler (version 7.3) will need to add another directory as follows:

setenv LD_LIBRARY_PATH $VJ_BASE_DIR/lib:/usr/lib32/cmplrs

Irix supports several Application Binary Interfaces (ABIs). VR Juggler supports the N32 and 64 formats. There are different library path variables for each:

ex. setenv LD_LIBRARYN32_PATH $VJ_BASE_DIR/lib32

ex. setenv LD_LIBRARY64_PATH $VJ_BASE_DIR/lib64

Optional VR Juggler-related Environment Variables

PATH

If you intend to use the VjControl program, you may want to add the $VJ_BASE_DIR/bin directory to your $PATH:

ex. setenv PATH $VJ_BASE_DIR/bin:$PATH

VJ_DEBUG_NFY_LEVEL

This variable can be used to control the amount of diagnostic information a VR Juggler application outputs. Its value is a number between 0 (only very important messages are printed) and 7 (vast amounts of data). Nonhackers are advised to use levels 0 through 3, as higher debug levels become increasingly cryptic and can severely impact application performance. The default is level 1 - only errors and critical information is output.

ex. setenv VJ_DEBUG_NFY_LEVEL 1

VJ_DEBUG_CATEGORIES

This variable can be used to control which components of VR Juggler are allowed to output diagnostic data. If for some reason you set VJ_DEBUG_NFY_LEVEL to 5 or higher, this variable can be used to filter the output.

The value of VJ_DEBUG_CATEGORIES is a space-separated list of Juggler debug component names (defined in $VJ_BASE_DIR/include/Kernel/vjDebug.h). The default value is "DBG_ALL", which performs no filtering whatsoever.

ex. setenv VJ_DEBUG_CATEGORIES DBG_ERROR

ex. setenv VJ_DEBUG_CATEGORIES "DBG_KERNEL DBG_INPUT"

ex. setenv VJ_DEBUG_CATEGORIES "DBG_CONFIGDB DBG_ENV_MGR"
