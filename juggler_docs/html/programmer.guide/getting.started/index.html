There a few things needed to get started using VR Juggler.

Required Reading:

  • A book on C++ and object oriented design
  • A book(s) about Juggler's supported graphics API(s). (OpenGL Red Book, Performer manual, etc...)

Environment Variables

The environment variable VJ_BASE_DIR must be set to the base directory of the installed Juggler library.  This tells the kernel and the config editor where to find data files needed to run the system.

Ex. setenv VJ_BASE_DIR /home/software/Juggler/beta-0.1

In order to run the Java based config editor, the environment variable JDK_HOME must be set to tell the system where Java is located.

Ex. setenv JDK_HOME /user/java

Search Path

The search path must have VJ_BASE_DIR/bin added to it in order for the java programs to be found correctly.  The following line will add the correct path to the user search path.

set path = ( $VJ_BASE_DIR/bin $path )

Linking to Dynamic Shared Objects

By default Juggler is setup to link to dynamic shared object libraries. (dso's) what this means is that instead of including all the juggler object code into each executable compiled, the Juggler object code is dynamically linked at runtime.  This greatly decreases the size of the executable files generated.

In order for dynamic linking to work, there are some environment variables that need to be set to tell the dynamic linker where to look for the dso's.

For N32: setenv LD_LIBRARYN32_PATH $VJ_BASE_DIR/lib32
For 64: setenv LD_LIBRARY64_PATH $VJ_BASE_DIR/lib64

Once these variables are set, the dynamic linker will be able to find where the Juggler libraries are located.

Config files

Juggler requires config files in order to setup the system.  In the current version of the library, config files are passed to the application as a parameters.  By default, these config files are located in the users home directory in the ~/.vjconfig subdirectory. 

Each release starting with alpha 4 includes sample config files that may be copied to the users ~/.vjconfig directory to be used as a starting point for customization.   

Sample config files

The sample files are located in VJ_BASE_DIR/Data/configFiles.   (Where VJ_BASE_DIR is the base directory for the Juggler release being used.) 

In order to get started using Juggler quickly, you should copy all the config files from the configFiles directory into your local ~/.vjconfig directory.

NOTE: You are not required to use the default configuration files.  You are encouraged to customize them to suit your individual tastes and needs.

Sample code

There are sample applications located in the "test" directory in the installed Juggler area (VJ_BASE_DIR). Users can copy these files into there local home directories in order to compile them.  Each application is in a separate directory and has a makefile that will compile the application.

NOTE: Currently there are also internal test programs in the test directory.  In the future there will be a samples directory with only sample applications.  The directories that contain sample applications are:

  • ogl/wand
  • ogl/cubes
  • ogl/simpleGlove
  • ogl/torus
  • ogl/glove
  • ogl/combo
  • pfNav

Executing the test code

All of the test programs take a config file as a parameter.  For example to run the cubes application a user would type:

cubes ~/.vjconfig/sim.base.config ~/.vjconfig/sim.wand.mixin.config

Some of the applications need access to VJGlove, which is an alias for a glove device.  For these programs you also need to specify the ~/.vjconfig/sim.glove.mixin.config file.

If the program complains about not being able to find the runtime libraries it needs, see the section above about dynamic shared objects.
