Linking to Dynamic Shared Objects

By default Juggler is setup to link to dynamic shared object libraries. (dso's) what this means is that instead of including all the juggler object code into each executable compiled, the Juggler object code is dynamically linked at runtime.  This greatly decreases the size of the executable files generated.

In order for dynamic linking to work, there are some environment variables that need to be set to tell the dynamic linker where to look for the dso's. These are called LD_LIBRARY paths.

IMPORTANT!    Be sure to set up your LD library paths, or Juggler may not run (even though it will compile!)
Click the paths topic on your left.

Once these variables are set, the dynamic linker will be able to find where the Juggler libraries are located.
