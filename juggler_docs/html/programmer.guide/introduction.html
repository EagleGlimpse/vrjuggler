Introduction

Welcome

Welcome to the VR Juggler programmers guide.

VR Juggler applications are not difficult to write once you understand the basics.  The goal of this book is to introduce the basics that are needed and to give you experience along the way with hands-on exercises.

Why should you read this book?

Hopefully you are reading this book because you want to create cool and innovative new VR applications that push the limits of today's technologies.  VR applications run the gamut from real-time entertainment applications to life saving medical applications.  The sky is the limit.

There are two groups of people who are probably reading this book.

First, there are those of you that are required to read it to do a project for work.  To those of you in this category, I just want to say don't worry.  VR Juggler is actually very simple to use after you get through the initial learning stages.  it is a very powerful tool that will allow you to create interesting and powerful application very quickly.

The other group of readers are those of you that are just interested in creating cool VR applications.  I can happily tell you that you have come to the right place.  VR Juggler will allow you to create extremely powerful applications very quickly that will run on nearly any hardware platform you would like. 

Experience needed

Before reading this book, you should have already completed the getting started book and be able to:

  • Compile, configure, and run a VR Juggler sample application

Required background

To get the most out of this book, there are a few prerequisites.

  • Programming experience using C++
  • Some graphics programming background (OpenGL, Performer, etc)
  • Reasonable mathematical background.

For some of the advanced sections of this guide, it is recommended that you have read and understand the VR Juggler Architecture guide book as well.  Don't worry to much though because in places where this information is necessary we will provide cross-references to the appropriate information.

What if have used another VR software system?

If you already have experience with other VR software development environments, you can easily skim through this book and cull out the relevant new information.  The book is designed for easy skimming.  Just look at the headings and you should get a very good determination of what you should read and what you can skip over.  I would highly recommend examining all of the tutorials in detail.  Then if there is something in the tutorial that you do not understand, you can backtrack to find the appropriate information.

How should you read this book?

This book is designed to allow reading from cover to cover.  Each chapter builds on the information from the chapters before and expands the readers working knowledge of VR Juggler.

The book can also be used as a reference for developers that have already acquired some general VR Juggler experience.

Book Organization

The programmer's guide is organized into 4 main sections.  These sections are described below.

  • Part I: Application basics
     In this section you will be introduced to the key application development concept of VR Juggler: application objects.  The information is needed in order to understand the rest of the book, so it is highly advisable to read this chapter even if you have experience with other VR software development environments.
     
  • Part II: Common helper classes
    This chapter describes some of the key classes that all applications make use of.  These classes are used for math, input, etc.
     
  • Part III: Writing applications
    This section comprises the main bulk of information about application development.  It shows how to get input from the system and how to write applications for each of the currently supported graphics APIs.
     
  • Part IV: Advanced Topics
    This section covers advanced topics that are not covered in the other sections.  Included in these are the topics of: run-time reconfiguration, thread creation, and multi-threaded application development.
