Juggler base application construction

Application object

As was described in previous sections (see app object), all VR Juggler applications derive from a base VR Juggler application object class (vjApp).  This class defines the basic interface that VR Juggler expects from all application objects.

This means that when constructing an application, the user defined application object must inherit from a class either vjApp or a draw manager specific application class that has vjApp as a superclass.

For example:

class userApp : public vjApp
{
public:
   init();
   preFrame();
   postFrame();
}

This defines a new application class (userApp) that can be used anywhere that VR Juggler expects an application class.

Draw manager specific application classes

A user application does not have to (and in most cases does not) derive from vjApp.  In most cases, an application class is actually derived from a draw manager specific application class.

For example:

class userGlApp : public vjGlApp
{
public:
   init();
   preFrame();
   postFrame();
   
   draw();
}

This user application is an OpenGL application.  The application class (userGlApp) has derived directly from the OpenGL draw manager specific vjGlApp application base class.  This class provides extra definitions in the interface that are custom for OpenGL applications.

See base VR Juggler interface for more details.
