Getting input from VR Juggler

How does an app get input?

There are many types of input devices that VR Juggler applications can use, positional, digital, analog, and quite a few more.  But the process and concepts about how to get input from all these devices is exactly the same in all applications.

The primary thing to remember about getting input in applications is that all VR Juggler applications receive input through device handles that are managed by vjDeviceInterfaces .  There are vjDeviceInterfaces for each type of input data that VR Juggler can handle.  There is one of positional input, one for analog, and so on.  The all have very similar interfaces and behave exactly the same.  (please see vjDeviceInterface for details)

Show me the code

So, what does the code look like for getting device input using these device interfaces.  Since all device interfaces look the same, I will just give an example of getting positional input.  All other types are very similar.

class myApp : public vjApp
{
public:
   init();
   preFrame();
private:
   vjPosInterface mWand;
}

The first thing that has to be added to an application is the variable that is the actual device interface. Usually they are member variables of the user application class.  This is how mWand is declared above. 

myApp::init()
{
   mWand.init("NameOfPosDevInConfiguration");
}

The device interface has to be told what device it is getting data from, this is done by calling the init() function with the string name of device that the interface should get data from. This device name comes from the active configuration.

So, now we know how to declare a device interface and how to initialize it. How do we actually use it??? 

...
vjMatrix wand_pos;
wand_pos = *(mWand->getData());
...

This code shows an example of using the device interface in an application.  It shows some sample code where the application copies the positional information from a device interface.  When it is dereferenced, the device interface figures out what device it is pointing to and returns the data from that device.

Once again, please see vjDeviceInterface for much more complete details on how to use vjDeviceInterfaces.
