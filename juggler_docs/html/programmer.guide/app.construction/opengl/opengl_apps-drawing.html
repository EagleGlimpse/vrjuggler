OpenGL drawing - vjGlApp::draw()

The most important (and visible) component of most OpenGL applications is the OpenGL drawing.  The vjGlApp class interface defines a draw() member function to hold the code for drawing a virtual environment.

Any OpenGL drawing calls should be placed in the vjGlApp::draw() function of the user application.  Adding drawing code to an OpenGL base VR Juggler application is a straight forward.

This function is called whenever the OpenGL draw manager needs to render a view of the virtual world created by the user's application.  It is called for each defined OpenGL context, and it may be called multiple times per frame in the case of multi-surface setups and/or stereo configurations.

Applications should never rely upon the number of times this function is called per frame.

When the function is called, the OpenGL model view and projection matrices have been configured correctly to draw the scene.

Input devices are guaranteed to be in the same state (position, value, etc) for each call to this function for a given frame.

What should you do here

The only code that should execute in this function is calls to OpenGL drawing routines.  It is permissible to read from input devices to determine what to draw, but application data members should not be updated in this function.

What should you not do here

This function should not be used to perform any time consuming computations.

Code in this functions should not change the state of any application variables.
